# Infrastructure Configuration Guide

## Connecting Custom Domain (Spaceship to Vercel)

This outlines how `joinstrategicpathways.com` is configured to route traffic
from the Spaceship domain registrar to the Vercel hosting environment.

### 1. Vercel Configuration

1. In Vercel Project Settings > Domains
2. Add `joinstrategicpathways.com` and `www.joinstrategicpathways.com`
3. Environment: **Production**
4. Redirects: **No Redirect** (Let Vercel automatically handle routing `www` to
   the root domain).

### 2. Spaceship DNS Panel

In the Advanced DNS settings for `joinstrategicpathways.com` on Spaceship, the
following records route the web traffic:

| Type         | Host/Name  | Value                  | Purpose                                       |
| :----------- | :--------- | :--------------------- | :-------------------------------------------- |
| **A Record** | `@` (Root) | `76.76.21.21`          | Points the apex domain to Vercel's Anycast IP |
| **CNAME**    | `www`      | `cname.vercel-dns.com` | Aliases the `www` subdomain to Vercel         |

_Note: Delete any default Spaceship parking `A` or `CNAME` records to avoid
conflicts._

## Authenticating Email Delivery (Spaceship to Resend)

To send transactional emails from `@joinstrategicpathways.com` without being
flagged as spam, the domain must be cryptographically authenticated using
Resend.

### 1. Resend Configuration

1. In Resend Dashboard > Domains
2. Add `joinstrategicpathways.com` (Region: `us-east-1`)

### 2. Spaceship DNS Panel

Add the following records generated by Resend to Spaceship's Advanced DNS:

| Type    | Host/Name           | Value                                   | Priority | Purpose                                                  |
| :------ | :------------------ | :-------------------------------------- | :------- | :------------------------------------------------------- |
| **MX**  | `@`                 | `feedback-smtp.us-east-1.amazonses.com` | `10`     | Handles bounce notifications                             |
| **TXT** | `resend._domainkey` | `p=...` _(Long public key from Resend)_ | N/A      | **DKIM**: Cryptographically signs emails                 |
| **TXT** | `@`                 | `v=spf1 include:amazonses.com ~all`     | N/A      | **SPF**: Authorizes Resend/Amazon to send on your behalf |

### 3. Environment Variable

Once Resend verifies the DNS records (can take up to 48 hours to globally
propagate):

1. Generate an API Key in Resend with "Full Access" (starts with `re_`).
2. Add the key to Vercel Project Settings > Environment Variables as
   `RESEND_API_KEY`.
